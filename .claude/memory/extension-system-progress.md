**关键非显然决策**：
1. **Ext Host 的 RPC 对端是 renderer 不是 main**——main 只搬 stdio 字节（同 ACP）。命令路由目标 CommandsRegistry / View UI / SCM 状态都在 renderer。
2. **stdio 分帧用换行分隔 UTF-8 帧，不用 base64**——因为 ChannelServer/Client 的帧是 `TextEncoder().encode(JSON.stringify(msg))`，JSON 把字符串内换行转义为 `\n`(两字符)，编码字节绝不含裸 0x0A；`setEncoding('utf8')` 保证不在多字节中间断开。
3. RPC 复用 platform 现成 `ChannelServer/ChannelClient/ProxyChannel`，唯一新代码是 stdio 适配器。ProxyChannel 约定：`/^on[A-Z]/` 是事件，其余方法名直传 call。
4. extensions-common 需 `@types/node` + tsconfig `types:["node"]`（用了 TextEncoder）。apps/editor 把 extensions-common 放 **devDependencies**（同 platform/workbench-ui，避免打包崩，见 [[electron-builder-asarunpack-pnpm-workspace]]）。
5. **API 注入用 globalThis bridge 而非 ESM loader hook**：extension-api 是委托到 `globalThis.__universeExtensionHostBridge__` 的薄 shim，**bundle 进每个扩展**（git esbuild 去掉 external）。host 在 import 扩展前 `installApiBridge(service)` 装好全局 bridge。BRIDGE_KEY 在 extension-api 与 apiFactory 各写一份(注释标注 KEEP IN SYNC)。比拦截裸 import 简单且健壮。
6. **命令面板直接读 `CommandsRegistry.getCommands()`**（非 MenuId.CommandPalette），翻译只需注册到 CommandsRegistry + metadata{description=title, category}。MainThreadCommands 对已存在命令(manifest bootstrap proxy)不覆盖，避免 metadata 退化。
7. **bootstrap proxy 直接调 `client.executeContributedCommand` 不经 CommandService 再分发**，避免激活后真 handler 未注册时无限循环。
8. **extension-api 的 `StatusBarAlignment` 必须是普通 `enum` 而非 `const enum`**：git 扩展 tsconfig 开 `isolatedModules`，跨模块访问 ambient const enum 报 TS2748。普通 enum 生成运行时对象，bundle 进扩展即内联。platform 内部的 const enum 不受影响（renderer 由 vite 打包，非 isolatedModules）。
9. **SCM 视图是内置宿主组件（ScmView），不是 manifest 贡献的 view**——对标 VSCode：SCM viewlet 内置，扩展只通过 `scm.createSourceControl` 注册 provider。因此 Phase 4 不做 manifest views/viewsContainers 翻译（推迟到 Phase 6 通用 view 贡献）。SCM 容器/视图走内置 BuiltIn{ViewContainers,Views}Contribution（套路 B），componentKey=`scm.main`。
10. **SCM 句柄全局唯一且单调递增**——source control 与 group 共用 ext host 的一个计数器（`_scmHandle++`），所以 renderer 用扁平 `Map<groupHandle,{sc,group}>` 即可路由 resource 更新，无需遍历 provider。inputBox 双向：host set value→`$setInputBoxValue`→renderer textarea；用户输入→`changeInputBoxValue`→`$onInputBoxValueChange`→host `acceptRendererValue`（更新 _value + fire onDidChange，**不回 echo**避免环）。
11. **行内/标题菜单动作的 `when` 在 ScmView 内用 `ContextKeyExpr.deserialize(when).evaluate(ctx)` 求值**，ctx 是合成 `{getValue:(k)=>scope[k]} as IContext`，scope={scmProvider, scmResourceGroup, scmResourceState(=contextValue)}。注意 `IMenuItem.when` 可能是 string 也可能已是 `ContextKeyExpression`（registerAction2 path），evalWhen 要两种都吃。
12. **workspace 路径用 env 传给 ext host（不新建 RPC channel）**：`start(spec?)` → main 把 `spec.workspaceRoot` 写进 `UNIVERSE_WORKSPACE_ROOT`（仿 `UNIVERSE_BUILTIN_EXTENSIONS_DIR`，survives denylist）→ bootstrap 读 env → `ExtensionService.getWorkspaceRoot()` → `workspace.rootPath`。**静态**：启动时固定；运行时切 workspace 不更新 ext host（Phase 6 重启）。renderer `ExtensionHostClientService` 必须 `await IWorkspaceService.whenReady` 再读 `current?.folder.fsPath`，否则抢在跨进程 hydration 前 spawn 拿不到。`_workspaceRoot` 设为构造**可选**参数，免改全部 extensionService.test。
13. **git 全程用 CLI**：`gitExec` = `spawn('git', argv数组)`（不拼 shell 防注入）+ 剥 env denylist（与 host 同款）+ Buffer 收集后 `toString('utf8')`（保住 `-z` 的 NUL，文件名不被破坏）。porcelain v2 的 rename(`2 `)条目占**两个** NUL 字段（新路径 + 原路径），parser 要多吃一个 token。`refresh` 重入安全：进行中再来只置 `_queued`，完成后补跑一次。watcher 用 `fs.watch(root,{recursive:true})`（win/mac 支持，linux 不支持时 catch fallback 到 watch `.git`），`.git` 内仅 index/HEAD 触发刷新、其余（objects/logs）忽略，debounce 400ms。
14. **双 host 信任级隔离（Phase 6）**：main 端 `Map<handle,ProcEntry>` 本就支持多进程，单 host 假设只在 renderer。抽 `HostConnection`（一连接=一 protocol+client+server+全部 MainThread* 通道），`ExtensionHostClientService` 持 `_trusted`/`_restricted` 两个。**受限 host 不给 SCM**（HostConnection 无 deps.scm + ExtensionService kind 门控）。命令路由：runtime 命令各连接自带 MainThreadCommands 闭包自己的 extHost proxy（天然正确）；**静态贡献命令需账本** `_commandOwner: Map<id,conn>`（getContributions 时记账 + MainThreadCommands ledger 回调），因为静态命令的 bootstrap proxy 调的是 client service 的 executeContributedCommand。host→renderer 执行命令（真 Diff 用）经 `IMainThreadCommands.$executeCommand`，**只放行 `_workbench.*` 前缀**防 host↔renderer 环；ext host 的 executeCommand 本地表 miss 才转发。
15. **fs 网关与隔离强度的诚实边界**：`workspace.fs` 全走 `MainThreadFs`（renderer）→ 复用 `AcpPathPolicy` + `IFileService`；wire 上 content 用 base64（newline-JSON 不能传 Uint8Array）。**真正的 OS 级 fs 锁定（Node `--experimental-permission`）改 opt-in**（`UNIVERSE_EXT_HOST_PERMISSION=1` + spawnSync 探测），默认软隔离——因为窄 `--allow-fs-read` 可能让 Electron-as-node 读不到自身资源而启动崩溃循环，且 Electron 在 ELECTRON_RUN_AS_NODE 下是否支持该模型本机不可验证。受限 host **无外部扩展时不 spawn**（main `hasUserExtensions()` 探测 `<userData>/extensions`）。
