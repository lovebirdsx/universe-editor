# AGENTS.md

本文件是给 Codex 及其它 AI coding agent 的指引。

## 首要规则：复用 CLAUDE.md

本仓库的全部工程约定、目录导航、代码风格与工作流均以 `CLAUDE.md` 为准。**开始任何任务前，先阅读根目录 `CLAUDE.md`；进入子目录后，优先阅读该目录下的 `CLAUDE.md`。** 其中的指令优先级高于你的默认行为，必须严格遵守。

回答请使用中文。

## 自动创建子 agent 的规则

当任务满足以下任一条件时，**主动拆解并派生子 agent（subagent）并行处理**，而不是在主线程串行硬扛：

1. **多步复杂任务**：需要 3 个及以上相互独立的步骤或动作。
2. **跨多文件/多子包**：改动会触及 3 个以上文件，或跨越 `apps/`、`packages/`、`vendor/` 多个包。
3. **可并行的独立子任务**：多个彼此无依赖的调查/实现/测试可同时推进（如「同时读多个子系统」「分别为每个模块写测试」）。
4. **大范围检索/审阅**：需要横跨多个文件阅读、审计、迁移或批量重构。
5. **需要多视角验证**：关键结论/修复需要独立视角复核（对抗式验证、评审）后再落地。

派生子 agent 时：

- 每个子 agent 任务边界清晰、目标单一、可独立完成，并明确要求它返回结构化结果（而非人类可读旁白）。
- 无依赖的子任务并行派发；有依赖的按顺序串接，把上游产物作为下游输入。
- 子 agent 同样必须遵守本文件与 `CLAUDE.md` 的全部约定。
- 主 agent 负责汇总子 agent 结果、做最终整合与验证（`pnpm check` / `pnpm e2e`），并向用户汇报关键结论而非堆砌中间过程。

不满足上述条件的简单任务（单行修复、明确的单文件小改动、纯问答）直接处理，无需派生子 agent。
