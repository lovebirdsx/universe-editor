## 两套模型体系（重点）

编辑器里有两套模型，用途不同，别把它们搞混——改了一个不会影响另一个：

| | Agent 模型 | 平台 AI 模型 |
| --- | --- | --- |
| 作用对象 | 当前 Agent 会话 | 编辑器内置的 AI 功能 |
| 切换命令 | 选择 Agent 模型… | 选择 AI 模型 |
| 模型来源 | Agent 自身暴露的模型列表 | 你在设置里配置的供应商模型 |
| 不可用时 | 提示"活跃 Agent 未提供模型选择器。" | 需先配供应商，否则列表为空 |

> 提示："选择 Agent 模型…"要求当前有活跃会话、且该 Agent 提供了模型选择器，否则会弹上表中的提示。

## 接入第三方模型（Kimi、GPT 等）

内置的 Claude Agent 使用的是 Anthropic 的接口协议。要让它改用别的模型，关键不是随便填个模型名，而是把请求指向一个**兼容 Anthropic 接口的网关地址**。分两种情况：

| 目标模型 | 怎么接 |
| --- | --- |
| **Kimi K3**（Moonshot） | Moonshot 提供官方的 Anthropic 兼容端点，直接把网关地址指过去、填上它的 key 即可，无需自建代理。 |
| **GPT-5.6**（OpenAI） | OpenAI **不提供** Anthropic 兼容端点，需要自行搭一层协议转换代理（如 LiteLLM proxy、claude-code-router），把网关地址指向该代理。 |

配置步骤（在"打开 AI 与 Agent 设置" → Agents 分组 → Claude → **认证**）：

1. 点"添加凭据"，类型选 **自定义网关 / 认证 token**。
2. 填写：
   - **网关地址**（`ANTHROPIC_BASE_URL`）：Kimi 的兼容端点，或你的代理地址。
   - **认证 token**（`ANTHROPIC_AUTH_TOKEN`）：对应服务的密钥。
   - **模型**：向该网关请求的模型 id（例如 `kimi-k3`）。这一项会在你"使用"该凭据时一并写入模型设置——一个凭据就是一套"网关 + 模型"预设，一键切换。
   - **快速 / 后台模型**（可选，`ANTHROPIC_SMALL_FAST_MODEL`）：该网关提供的更小更快的模型。
3. 保存后点该凭据的"使用"，即完成切换。想切回 Claude 官方，就"使用"另一套凭据或你的 Claude 登录。

> 凭据可以存多套（Claude 官方、Kimi、GPT 代理各一套），随时一键切换，互不覆盖。带模型预设的网关凭据只有在**网关地址、token、模型**都与当前设置一致时才显示"使用中"。

## 按功能分配模型

编辑器的多个 AI 功能可以各用各的模型（例如内联补全用更小更快的模型省钱）。在"打开 AI 与 Agent 设置"的 Agents 分组里，可为四类功能分别指定模型：

| 功能 | 界面名 | 说明 | 单独命令 |
| --- | --- | --- | --- |
| 对话 | 对话 | AGENTS 会话与对话补全使用的模型 | 由 Agent 模型控制 |
| 提交信息 | 提交信息 | 生成 Git 提交信息使用的模型 | 选择提交信息模型 |
| 内联补全 | 内联补全 | 编辑器幽灵文本补全使用的模型（可选更小更快的） | 选择内联补全模型 |
| 会话标题 | 会话标题 | 为 AGENTS 会话生成友好标题使用的模型 | 选择会话标题模型 |

## 打开设置

从[命令面板](../reference/glossary.md#命令面板)搜"打开 AI 与 Agent 设置"即可进入上述配置界面。

## 看一次会话花了多少钱

- 输入框下方有一个 **¥ 人民币开销** 的小标签，悬浮时提示"会话费用 - 点击查看明细"。
- 点击它会弹出"会话费用明细"，按模型列出输入 / 输出 token 数、各模型开销和总计。
- 数据来自 Agent 上报的真实用量（含子任务开销）；无法精确统计时会标注为"预估会话费用"。

## 看上下文用量与自动压缩

对话越长，占用的上下文窗口越多。快满时 Agent 会**自动压缩**（把较早的对话摘要成更短的内容）以腾出空间。编辑器在两处给出提示：

- **发送按钮的圆环**：外圈进度弧表示当前上下文占用比例；悬浮可看具体的 token 用量。
- **接近阈值预警**：占用达到 **80%** 时，输入框上方会出现一个「上下文 XX% · 接近自动压缩」的提示条，达到 **90%** 时转为红色。未达阈值时不显示。
- **压缩状态卡片**：真正开始压缩时，时间线里会出现一张系统卡片——压缩进行中显示「正在压缩上下文…」，并带一条进度条与「XX% · 用时」实时提示（进度为按耗时估算，用于直观反馈进展）；完成后变为「上下文已压缩」并显示总用时，失败则显示失败原因。

> 压缩由 Agent 自动完成，你无需手动干预；压缩后发送按钮圆环会立即刷新为压缩后的真实占用。

压缩只影响 Agent 的工作记忆，不会丢失你的对话记录：重新打开发生过压缩的会话时，压缩前的完整对话会照常回放，压缩发生的位置显示一张「上下文已压缩」卡片。

## 看一次会话跑了多久

会话运行时间只累计 Agent **实际运行的净时长**，不包含挂起或等待你输入的时间。
