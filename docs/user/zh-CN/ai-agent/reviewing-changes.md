# 审阅并采纳 Agent 的改动

AI [Agent](../reference/glossary.md#智能体--agent) 会直接把改动写到你的磁盘上。这一页教你去哪看它改了什么、怎么看懂、以及怎么决定采纳还是回退——让"AI 帮我改文件"这件事变得安全可控。

## 先建立一个心智

一次会话里，Agent 可能改了多个文件。编辑器把这些改动聚合成一个叫**[会话更改](../reference/glossary.md#会话更改)**的集合——它是"这次会话到底动了什么"的总账，可以逐个文件看[差异](../reference/glossary.md#差异--diff)。

它独立于 Git 的工作区改动视图：会话更改只盯着"这一次 Agent 会话"的产物。

## 打开会话更改

- 命令："显示会话更改"（[命令面板](../reference/glossary.md#命令面板)可搜，无默认快捷键）。
- 或打开会话编辑器标题栏的 `...` 菜单，选择"显示会话更改"。
- 视图位于[主侧边栏](../reference/glossary.md#辅助侧边栏)的"会话更改"容器（图标 `diff`）。

如果这次会话没改任何文件，视图会提示"此会话尚未修改任何文件。"

## 列表视图与树视图

右上角工具栏可以切换两种呈现方式：

- **以列表查看**：平铺所有改动文件。
- **以树查看**：按目录层级组织，文件多时更清楚。

## 预览与钉住

- **单击**文件 = 预览（复用同一个预览标签，快速扫过多个文件而不堆一堆标签）。
- **双击**文件 = 钉住成独立标签。
- 每行还有"打开文件" / "打开预览"入口。

## 看懂 diff

<!-- 截图：某文件的 diff 预览 -->

- **左旧右新**：左边是改动前，右边是改动后。
- **按状态着色**：新增、修改、删除分别标色；删除的内容没有可点的新文件。
- **看什么**：重点确认"改动是否符合你的预期、有没有误伤到不该动的地方"。

## 采纳还是回退

这里有个关键分工需要说清：**会话更改视图本身只用来"看"，真正的动手操作在 Git 那边。**

- **采纳**：确认改得对，通常意味着接下来用 Git 把这些改动**暂存并提交**，让它们成为项目的一部分。
- **回退**：改得不对，用 Git **放弃更改**，或撤销某个文件。

暂存、提交、放弃这些具体操作，在 [会话更改（Git）](../git/session-changes.md) 一册里手把手讲。本页只负责让你"看懂 diff、做出决策"。

## 给自己上一道保险

> 建议：在让 Agent 做大改之前，先把项目纳入 Git、提交一次干净的状态。这样无论 Agent 改成什么样，你都能一键回退到干净起点。
>
> 具体做法见 [提交与 AI 提交信息](../git/commit.md)。
