# AI 供应商配置

这一页教你接入外部 AI 服务：添加供应商分组、填好连接信息、设置和管理 API 密钥，并讲清楚密钥是怎么被安全保管的。需要用 AI 功能的用户才需要读这一页。

## 两个入口

配置 AI 供应商有两个入口，改的是同一份数据：

- **图形界面**：命令面板运行「打开 AI 与 Agent 设置」，打开可视化的 AI 设置编辑器，添加分组、验证、管理模型都在这里。推荐日常使用。
- **JSON 文件**：命令面板运行「打开 AI 设置（JSON）」，直接编辑配置文件 `aiSettings.json`，适合高级用户。

按 `Ctrl+Shift+P` 打开[命令面板](../reference/glossary.md#命令面板)后，输入上面的命令名即可。

## 添加一个供应商分组

一个「供应商分组」就是一套 AI 服务的连接配置。添加步骤：

1. 在 AI 设置编辑器里点「添加提供方分组」。
2. 填写下面几个字段：

   | 字段 | 说明 |
   | --- | --- |
   | 名称 | 给这个分组起个好认的名字 |
   | 供应商 | 选择服务类型（例如 `openai`、`ollama`） |
   | Base URL | 服务地址，留空则用该供应商的默认地址 |
   | API 密钥 | 访问服务的密钥，可选（本地服务通常不需要） |

3. 点「验证」，编辑器会尝试连接并列出可用[模型](../reference/glossary.md#模型)的数量；连接失败会显示错误原因。

添加好的分组会显示徽标：有没有配密钥、枚举到多少个模型，一目了然。

## 管理分组里的模型

每个分组下面会列出自动枚举到的模型。你还可以：

- **手动添加模型**：填入该服务所需的模型 id，用于枚举不到但确实可用的模型。
- **配置模型**：调整单个模型的参数。
- **移除模型 / 移除分组**：删掉不再需要的条目。

## 接入 OpenAI 兼容端点

很多本地或第三方服务（如 LM Studio、vLLM、DeepSeek 等）提供「OpenAI 兼容」的接口。接入方法：

1. 供应商选 `openai`。
2. Base URL 填入该服务的兼容端点地址。
3. 如果该服务不需要认证，API 密钥留空即可（连接时会省略认证信息，不会报错）。

## 接入本地部署的 Ollama

如果你在本机跑了 Ollama：

1. 供应商选 `ollama`。
2. Base URL 通常无需填写，默认就是本机的 Ollama 地址 `http://localhost:11434`。
3. 本地服务无需 API 密钥。

## 设置与清除 API 密钥

除了在添加分组时填写，你随时可以单独管理密钥。

**在图形界面里**：进入某个分组的详情，用「设置 API 密钥」和「清除 API 密钥」按钮操作。密钥状态会显示为「已存储」或「未设置」。

**用命令面板**：

- 「设置 AI 提供方 API 密钥」：先选一个分组，再输入密钥。
- 「清除 AI 提供方 API 密钥」：选一个分组，确认后清除。

## 密钥安全：你需要知道的

API 密钥是敏感信息，编辑器对它做了严格保护。请记住这三点：

1. **加密存储在操作系统的安全存储里。** 你输入的密钥由操作系统提供的安全存储（safeStorage）加密保管，不是明文躺在某个文件里。设置密钥时的输入框也会提示：「加密存储，绝不写入 aiSettings.json」。
2. **绝不写入任何明文配置文件。** 密钥不会被写进 AI 设置文件、也不会写进普通的编辑器设置文件，任何配置文件里都不会出现你的密钥明文。
3. **不经界面进程明文传递。** 密钥只在后台主进程里加密保存，不会以明文形式在界面进程之间传输。

> 因此，**把 AI 设置文件 `aiSettings.json` 分享给别人，或提交到版本库，都是安全的**——它里面根本不含密钥。你完全不必担心分享配置会泄露密钥。
>
> 补充：如果操作系统的安全存储不可用，设置密钥的操作会直接报错失败，编辑器**不会**退而求其次地把密钥以明文存下来。

**请不要**把密钥手动写进任何设置文件——编辑器的设计不需要你这样做，也不支持这样做。始终用上面的界面或命令来管理密钥。

## 配置文件里有什么

如果你打开 AI 设置文件 `aiSettings.json`，会看到这些内容（都不含密钥）：

- `groups`：你添加的供应商分组，含名称、供应商、Base URL、模型列表。
- `activeModels`：当前为对话、内联补全分别选中的模型 id。

文件缺失或为空时，编辑器会自动合成一组默认分组（`ollama/default` 与 `openai/default`）。由于不含密钥，这个文件可以随[工作区](../reference/glossary.md#工作区--项目)一起提交到版本库。
