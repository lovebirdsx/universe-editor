## 什么是扩展

扩展是给编辑器增加额外能力的小模块。你不需要手动下载或启用它们——下面这六个都随编辑器一起装好，开箱即用。

## 六个内置扩展

### AI Assist

帮你借助 AI 更快地干活。目前最直接的一项是：在[源代码管理](../reference/glossary.md#源代码管理scm)面板的提交输入框旁提供一个图标按钮，点一下就能让 AI 根据你的改动自动写好一条提交说明。

<!-- 截图：SCM 输入框旁的 AI 生成提交信息图标 -->

### Claude Helper

帮你更顺手地编辑 Claude Code 的配置文件。当你打开 `.claude` 目录下的配置文件时，它会提供自动补全和检查，帮你避免写错字段。

### Git

帮你在编辑器里完成版本管理：提交、暂存、切换分支、推送、查看历史等等，一共提供五十多条 Git 相关命令。它是[源代码管理](../reference/glossary.md#源代码管理scm)面板背后的能力来源。详细用法见 [Git 概览](../git/overview.md)。

### Markdown Language Features

帮你更高效地编辑 Markdown 文档：在长文里快速跳到某个标题、在标题之间跳转、找出指向不存在位置的失效链接等。详细用法见 [Markdown 编辑](../editing/markdown.md)。

### Numbered Bookmarks

帮你在文件里放下 0 到 9 共十个编号书签，然后一键跳回。编辑时书签会跟着内容自动移动，行号旁还会显示书签图标，方便你在长文件里来回穿梭。详细用法见 [数字书签](../editing/bookmarks.md)。

### TypeScript Language Features

帮你更顺畅地阅读和编辑脚本文件（`.ts`、`.js` 等）：跳到定义处、查找引用、悬停查看说明、输入时补全、重命名，以及标出可能的错误。

## 为什么有些能力要打开对应文件才出现

为了不拖慢启动，扩展只在真正用得上的时候才「醒来」。所以你可能注意到某些能力要打开特定类型的文件后才出现：

| 扩展 | 什么时候开始工作 |
| --- | --- |
| Git | 编辑器启动完成后自动就绪 |
| Numbered Bookmarks | 编辑器启动完成后自动就绪 |
| TypeScript Language Features | 打开 `.ts`、`.js`、`.tsx`、`.jsx` 文件时 |
| Markdown Language Features | 打开 `.md` 文件时 |
| AI Assist | 首次使用「生成提交信息」时 |
| Claude Helper | 打开 Claude Code 配置文件时自动生效 |

> 提示：这就是为什么在纯文本文件里看不到脚本文件的跳转、补全——它们是脚本编辑专属的能力，打开对应文件后自然会出现。

## 关于安装第三方扩展

当前版本以内置扩展为主，**暂不支持安装第三方扩展**。上面这六个已覆盖大多数游戏内容创作场景。如果你有其它能力需求，欢迎通过反馈渠道告诉我们。
