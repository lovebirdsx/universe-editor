## 什么是扩展

扩展是给编辑器增加额外能力的小模块。你不需要手动下载或启用它们——下面这六个都随编辑器一起装好，开箱即用。

## 六个内置扩展

### AI Assist

帮你借助 AI 更快地干活。目前最直接的一项是：在[源代码管理](../reference/glossary.md#源代码管理scm)面板的提交输入框旁提供一个图标按钮，点一下就能让 AI 根据你的改动自动写好一条提交说明。

<!-- 截图：SCM 输入框旁的 AI 生成提交信息图标 -->

### Claude Helper

帮你更顺手地编辑 Claude Code 的配置文件。当你打开 `.claude` 目录下的配置文件时，它会提供自动补全和检查，帮你避免写错字段。

### Git

帮你在编辑器里完成版本管理：提交、暂存、切换分支、推送、查看历史等等，一共提供五十多条 Git 相关命令。它是[源代码管理](../reference/glossary.md#源代码管理scm)面板背后的能力来源。详细用法见 [Git 概览](../git/overview.md)。

### Markdown Language Features

帮你更高效地编辑 Markdown 文档：在长文里快速跳到某个标题、在标题之间跳转、找出指向不存在位置的失效链接等。详细用法见 [Markdown 编辑](../editing/markdown.md)。

### Numbered Bookmarks

帮你在文件里放下 0 到 9 共十个编号书签，然后一键跳回。编辑时书签会跟着内容自动移动，行号旁还会显示书签图标，方便你在长文件里来回穿梭。详细用法见 [数字书签](../editing/bookmarks.md)。

### TypeScript Language Features

帮你更顺畅地阅读和编辑脚本文件（`.ts`、`.js` 等）：跳到定义处、查找引用、悬停查看说明、输入时补全、重命名，以及标出可能的错误。

## 为什么有些能力要打开对应文件才出现

为了不拖慢启动，扩展只在真正用得上的时候才「醒来」。所以你可能注意到某些能力要打开特定类型的文件后才出现：

| 扩展 | 什么时候开始工作 |
| --- | --- |
| Git | 编辑器启动完成后自动就绪 |
| Numbered Bookmarks | 编辑器启动完成后自动就绪 |
| TypeScript Language Features | 打开 `.ts`、`.js`、`.tsx`、`.jsx` 文件时 |
| Markdown Language Features | 打开 `.md` 文件时 |
| AI Assist | 首次使用「生成提交信息」时 |
| Claude Helper | 打开 Claude Code 配置文件时自动生效 |

> 提示：这就是为什么在纯文本文件里看不到脚本文件的跳转、补全——它们是脚本编辑专属的能力，打开对应文件后自然会出现。

## 安装第三方扩展

除了内置扩展，你还可以安装第三方扩展来扩充编辑器的能力。有两条途径：从**扩展市场**在线搜索并一键安装，或者从本地的 `.vsix` 安装包安装（`.vsix` 是与 VSCode 兼容的扩展打包格式）。

### 从扩展市场安装

1. 按 `Ctrl+Shift+X`，或点击活动栏（最左侧的图标竖栏）里的**扩展**图标（拼图形状），或运行命令 **扩展: 显示已安装扩展**（英文 *Extensions: Show Installed Extensions*），打开扩展视图。
2. 在顶部搜索框里输入关键词，视图会分成「已安装」和「市场」两栏，市场栏列出匹配的可安装扩展。
3. 在某一项上点「安装」即可下载并安装；点击条目本身会打开详情页，可以查看说明文档（README）与它贡献的命令、配置、快捷键。

<!-- 截图：扩展视图的搜索结果与安装按钮 -->

> 发布版默认已接入官方扩展市场，打开扩展视图即可搜索安装。你也可以用启动参数 `--gallery-url` 或环境变量 `UNIVERSE_GALLERY_URL` 指向自建市场，或把地址置空以关闭市场（此时市场搜索为空，请改用下面的本地 `.vsix` 安装）。

### 从本地 VSIX 安装

1. 打开[命令面板](../reference/glossary.md#命令面板)（`Ctrl+Shift+P`），运行 **扩展: 从 VSIX 安装…**（英文 *Extensions: Install from VSIX…*）。
2. 在文件选择框里选中一个 `.vsix` 文件，点「安装」。
3. 安装完成后扩展立即生效——它贡献的命令会出现在命令面板里，随用随激活。

### 更新、禁用与卸载

- **检查更新**：运行 **扩展: 检查扩展更新**（*Extensions: Check for Extension Updates*）。编辑器会对照市场检查所有来自市场的扩展，有新版本时自动更新并提示更新了几个。
- **禁用与启用**：不想卸载、只想临时停用某个扩展时，可以禁用它。禁用后该扩展不再加载、不再贡献命令与界面。有两种作用范围：
  - **全局**：对所有窗口生效。在扩展视图里把鼠标移到条目上，点右侧的齿轮按钮（或右键点条目）打开操作菜单，选「禁用 / 启用」，或运行 **扩展: 禁用扩展**（*Extensions: Disable Extension*）/ **扩展: 启用扩展**（*Extensions: Enable Extension*）。
  - **仅当前工作区**：只对当前打开的文件夹生效，换一个工作区就不受影响。需先打开一个文件夹，然后在条目的操作菜单（齿轮按钮或右键）里选「禁用（工作区）/ 启用（工作区）」，或运行 **扩展: 禁用扩展（工作区）**（*Extensions: Disable Extension (Workspace)*）/ **扩展: 启用扩展（工作区）**（*Extensions: Enable Extension (Workspace)*）。工作区设置优先于全局设置。
  - 内置扩展（Git、TypeScript 等）同样可以禁用；禁用后对应能力（如源代码管理、脚本语言智能）会随之关闭，条目上会标有「内置」标记提醒你。
- **卸载**：运行 **扩展: 卸载扩展…**（*Extensions: Uninstall Extension…*）从列表里选择要移除的扩展，或在扩展视图里从条目的操作菜单（齿轮按钮或右键）选「卸载」。内置扩展不能卸载，只能禁用。

### 扩展可以提供的自定义预览

有些扩展会为特定文件类型注册**自定义编辑器**——你双击该类型的文件时，打开的不是普通文本视图，而是扩展提供的图形界面。市场里的 **PDF Viewer** 扩展就是一例：装上后，双击 `.pdf` 文件会直接在标签页里渲染 PDF（基于 Mozilla pdf.js），可翻页、缩放、跳转。

另一例是 **Excel Viewer & Diff** 扩展：装上后双击 `.xlsx` / `.xls` / `.csv` 会在标签页里渲染表格；它还支持**表格对比**——在资源管理器里对一个表格文件点右键选「选择以进行比较」，再对另一个选「与所选项进行比较」（或直接选中两个表格文件后选「比较所选文件」），即可并排查看两份表格的单元格级差异（新增/删除/修改高亮，可按 sheet 切换、只看差异行）。这套比较菜单是资源管理器内置的通用入口——文本文件走普通文本 diff，表格文件则自动交给该扩展渲染。若同时装了 Git 或 Perforce 扩展，对表格文件执行「Open Changes」也会用这种表格 diff 对比当前版本与仓库中的版本。

这类界面运行在一个受限的内嵌页面（iframe）里：它只能加载扩展自己目录下、以及扩展显式声明的少数几个位置的资源，并受内容安全策略（CSP）约束。但要注意——**这不是等同于浏览器网页的强沙箱**，承载它的扩展进程本身仍拥有接近编辑器的权限（见下方安全边界）。

### 关于安全边界

> ⚠️ **安全提示**：第三方扩展会以**接近编辑器本身的权限**运行——它们能访问你的文件和网络。编辑器**不会**把扩展关进沙箱，**安装即意味着相当程度的信任**。
>
> 为此编辑器做了两层保护，但它们只是提醒、不能替你判断：
>
> - **发布者信任提示**：首次安装某个发布者的扩展时会弹出确认，问你是否信任该发布者；确认后同一发布者的后续扩展不再重复询问。
> - **恶意扩展拦截**：市场会标记已知的恶意扩展，编辑器拒绝安装它们；若某个已装扩展事后被标记为恶意，会在下次启动时自动禁用并通知你。
>
> 请只安装你信任来源的扩展。
