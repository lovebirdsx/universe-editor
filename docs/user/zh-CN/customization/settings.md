## 两种修改方式

编辑器的设置可以用两种方式修改，改的是同一份数据，随你顺手：

- **图形界面**：带搜索栏和分类导航，看得见每一项的说明，适合日常调整。
- **JSON 文件**：直接编辑名为 `settings.json` 的文本文件，适合批量修改或复制粘贴分享配置。

## 打开设置

有三条路都能打开设置：

1. 按 `Ctrl+,` 直接打开图形界面设置编辑器。
2. 按 `Ctrl+Shift+P` 打开[命令面板](../reference/glossary.md#命令面板)，输入「打开设置」。
3. 从菜单栏「文件 → 首选项」里选择对应条目。

想直接编辑 JSON，在命令面板里搜「打开设置 (JSON)」即可。

## 图形界面设置编辑器

设置编辑器打开后，你会看到三个常用区域：

- **顶部搜索栏**：输入关键词过滤设置项，只显示相关的条目。
- **左侧分类导航**：按功能分组，点一下就跳到对应区块。
- **来源标记**：已经改过的项旁边会标出它的来源——默认、用户、工作区，或从 VSCode 沿用的用户 / 工作区设置。这样你一眼就能看出某个值是从哪里来的。

## 用户设置 vs 工作区设置

同一个设置项可以有两个层级的值。理解它们的优先级，能避免「我明明改了怎么没生效」的困惑：

| 层级 | 生效范围 | 存放位置 | 优先级 |
| --- | --- | --- | --- |
| 用户设置 | 你打开的所有[工作区](../reference/glossary.md#工作区--项目) | 用户配置目录 | 低 |
| 工作区设置 | 仅当前打开的文件夹 | 当前工作区内 | 高（覆盖用户设置） |

- 设置编辑器顶部有「用户」「工作区」两个标签页，切换即可分别编辑。
- **工作区设置需要先打开一个文件夹**才能编辑；没打开文件夹时，会提示「打开文件夹后才能编辑工作区设置。」
- 想直接编辑工作区的 JSON，在命令面板里搜「打开工作区设置」或「打开工作区设置 (JSON)」。

<!-- 截图：用户设置 vs 工作区设置切换标签 -->

> 提示：当同一项在两个层级都有值时，工作区设置说了算。这让你可以为某个具体项目定制专属配置，而不影响其它项目。

## 直接编辑 JSON

在命令面板里运行「打开设置 (JSON)」，会打开你的用户设置文件 `settings.json`。首次打开时，编辑器会自动写入一段带注释的模板，照着注释填就行。

它是纯文本的键值结构，例如：

```json
{
  'workbench.colorTheme': 'dark',
  'workbench.language': 'zh-CN'
}
```

**保存即生效，无需重启**（切换界面语言是例外，详见[主题与语言](./themes-and-language.md)）。

> 提示：界面语言其实也是一个普通的设置值（键名 `workbench.language`）。凡是能在图形界面里调的项，都能在这个 JSON 文件里找到对应的键。

## 从 VSCode 沿用设置

如果你以前用过 VSCode，可以直接读取它的设置文件：

- 命令面板运行「打开 VS Code 设置 (JSON)」，会打开 VSCode 的设置文件。
- 若你没装过 VSCode 或该文件不存在，编辑器会给出提示。
- 沿用过来的值，会在图形界面的来源标记里显示为「VSCode 用户」或「VSCode 工作区」，方便区分。

## 配置目录管理

「配置目录」是存放你的设置文件和快捷键文件（`settings.json` 与 `keybindings.json`）的文件夹。你可以把它挪到别处，例如放进云同步盘，在多台机器间共享同一套配置。

- **设置配置目录…**：选一个文件夹作为新的配置目录，切换时可以选择是否把当前设置复制过去。
- **打开配置目录**：在系统文件管理器里打开当前配置目录，方便你直接看文件。
- **将配置目录重置为默认值**：还原到默认位置。

> 注意：如果配置目录被命令行参数或环境变量锁定了，上面三个命令都会提示无法更改。这通常出现在受管理的环境里。
