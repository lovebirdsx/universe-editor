# HTML 预览

`.html` 文件默认以源码文本编辑器打开，方便你编写。想看渲染效果时，用一条命令在内嵌的浏览视图里预览整页——样式、图片、脚本都会像在浏览器里一样生效。

## 打开预览

以下命令仅当当前文件是 `.html`（含 `.htm`、`.xhtml`）文件时可用：

- **打开预览（替换当前标签）**：`Ctrl+Shift+V`，或运行"打开预览"。当前源码标签被预览替换。
- **在侧边打开预览（保留源码）**：`Ctrl+K Ctrl+V`，或运行"在侧边打开预览"。左边写源码、右边看渲染效果，边写边看。
- **从预览切回源码**：在预览标签上方点"打开源文件"按钮；预览获得焦点时也可以再按一次 `Ctrl+Shift+V`。

## 本地资源的加载范围

预览会加载页面引用的相对路径资源（`./style.css`、`<script src="app.js">`、`<img src="pic.png">`）以及网络资源（`http(s)`）。出于安全，只有**工作区内**、或与该 HTML 文档**同目录（及其子目录）**的本地文件才会被加载。

> 提示：预览基于文件本身的位置解析相对路径，因此以文档根为基准的绝对路径（如 `/app.js`）不一定能命中你预期的文件。需要这种绝对路径的项目，建议自行起本地服务器再在外部浏览器查看。

## 自动刷新

保存源文件，或该文件在磁盘上被外部改动时，预览会自动重新加载，无需手动刷新。

## 关于脚本执行

预览会**执行页面里的 JavaScript**，所以按钮、动画等交互都能正常工作。这意味着预览的能力接近真实浏览器——只预览你信任的 HTML 文件。
