# 内联补全（幽灵文本）

内联补全会在你打字时，于光标处以灰色"幽灵文本"实时给出续写建议，按 `Tab` 就能采纳。它比对话式 AI 更轻量，适合边写边补，加速文本输入。

## 开始前：先配好模型

> **重要**：内联补全要能工作，需要先在设置里配好一个可用的[模型](../reference/glossary.md#模型)。没配好时，手动触发会弹出提示引导你去选模型。
>
> 配置方法见 [AI 供应商配置](../customization/ai-providers.md)。

## 手动触发补全

正常打字时补全会自动冒出来。想在某处主动求一条建议，按 `Alt+\`，或运行命令面板里的"触发内联补全"。当前活动文件需要是可编辑的文本文件。

## 接受补全

看到中意的灰色幽灵文本，按 `Tab` 采纳它，建议就会变成正式内容。

## 跳到下一处建议

有时建议出现在离光标稍远的位置。此时按 `Tab` 会先把光标跳到建议所在处（而不是直接采纳）；到位后再按一次 `Tab` 才采纳。

## 开关内联补全

不想被幽灵文本打扰时，运行"开关内联补全"关掉它，之后就不再产生建议。再运行一次即可重新开启。

## 选择补全模型

运行"选择内联补全模型"，可以给补全单独指定一个模型。补全讲究快，通常挑一个比对话更小更快的模型体验更好——正如设置里的说明所言："编辑器幽灵文本补全使用的模型（可选更小更快的模型）"。

## 本页涉及的命令

| 命令面板名称 | 快捷键 | 作用 |
| --- | --- | --- |
| 触发内联补全 | `Alt+\`（编辑器文本焦点） | 主动求一条建议 |
| 采纳内联补全 | `Tab`（幽灵文本可见时） | 采纳当前建议 |
| 跳到下一处编辑建议 | `Tab`（光标不在建议处时） | 先跳到建议位置 |
| 开关内联补全 | 无 | 开启/关闭幽灵文本 |
| 选择内联补全模型 | 无 | 为补全指定模型 |
