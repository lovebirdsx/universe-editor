## 什么是 Markdown

Markdown 是一种轻量的标记语言：用 `**粗体**` 写粗体、用 `# 标题` 写标题、用 `- 列表` 写列表，纯文本就能排出结构清晰的文档。Universe Editor 能把这些标记即时渲染成漂亮的预览，写和看两不误。

## 打开预览

预览把源码渲染成最终效果，方便你确认排版。以下命令仅当当前文件是 `.md` 文件时可用：

- **打开预览（替换当前标签）**：`Ctrl+Shift+V`，或运行"打开预览"。当前源码标签被预览替换。
- **在侧边打开预览（保留源码）**：`Ctrl+K Ctrl+V`，或运行"在侧边打开预览"。左边写源码、右边看预览，边写边看。
- **从预览切回源码**：在预览标签上方点"打开源文件"按钮；预览获得焦点时也可以再按一次 `Ctrl+Shift+V`。

预览会渲染文档里的图片，包括网络图片（`http(s)`）、内嵌的 base64 图片，以及**本地图片**——用相对路径（`![图](./assets/图.png)`）或绝对路径都行，相对路径按"文档所在目录 → 工作区根目录"的顺序查找。出于安全，只有工作区内、或与文档同目录（及其子目录）的图片才会加载。

## 格式命令

以下命令都需要**编辑器文本获得焦点**、且文件语言是 Markdown。选中一段文字再按，会给这段套上对应格式；没选中时会插入标记。

| 操作 | 快捷键 | 命令面板名称 |
| --- | --- | --- |
| 加粗 | `Ctrl+B` | 加粗 |
| 斜体 | `Ctrl+I` | 斜体 |
| 行内代码 | `Ctrl+M` | 行内代码 |
| 删除线 | 无 | Toggle Strikethrough（切换删除线） |
| 数学公式 | `Ctrl+Shift+M` | 数学公式 |
| 升级标题 | `Ctrl+Shift+]` | Increase Heading Level（升级标题） |
| 降级标题 | `Ctrl+Shift+[` | Decrease Heading Level（降级标题） |
| 切换任务勾选 | `Alt+C` | Toggle Task Completion（切换任务勾选） |
| 格式化表格 | `Ctrl+Alt+T` | Format Table（格式化表格） |
| 整理链接定义 | 无 | Organize Link Definitions（整理链接定义） |

> 注意：上表中标了英文名的命令，目前在命令面板里显示的是英文原文（括号里是中文释义，方便你对照），后续本地化完成后会更新为中文。

"格式化表格"能把你手敲得歪歪扭扭的表格自动对齐成整齐的竖线，写表格时很实用。

## 拖入和粘贴文件、图片

在 Markdown 文件里插链接、贴图片，不用手打路径——**按住 `Shift` 把文件或图片拖进编辑器正文**，或直接**粘贴**，编辑器会自动生成对应的 Markdown 语法：

- **`Shift` + 拖入文件**：按住 `Shift`，从资源管理器或系统文件夹把文件拖到 Markdown 正文里。图片会生成 `![alt text](相对路径)`（渲染成图片），其它文件生成 `[text](相对路径)`（普通链接）。路径会尽量写成相对当前工作区的样子。
- **`Shift` + 拖入截图或图片 / 粘贴图片**：按住 `Shift` 从截图工具、浏览器网页拖来的图片（本地没有对应文件），或直接 `Ctrl+V` 粘贴，会自动保存到当前 `.md` 文件旁边的 `assets/` 文件夹里（文件名带时间戳），再插入 `![alt text](assets/图片名)`。
- **粘贴网址套住选中文字**：先选中一段文字，再粘贴一个网址（`https://…`），会生成 `[选中的文字](网址)`。

> 生成链接后，方括号里的占位文字（链接是 `text`、图片是 `alt text`）会**自动处于选中状态**，直接敲字即可替换成你要的名称，和 VSCode 一样。多个文件一起拖入时，可按 `Tab` 在各段占位文字间依次跳转填写。

> 为什么要按 `Shift`？**不按 `Shift`** 直接把文件拖进正文，仍是原来的行为——把它当成文件**打开**（拖到标签栏也一样打开成新标签）。只有**按住 `Shift`** 拖入才改成在光标处插入链接。这跟 VSCode 一致，避免和平时"拖动打开文件"的习惯冲突。粘贴图片则不需要按 `Shift`。

## 重命名文件时自动更新链接

在资源管理器里**重命名或移动**一个文件、文件夹后，其它 Markdown 文档里指向它的链接（以及被移动的 `.md` 文件自身内部的相对链接）会自动改成新路径，不用你手动逐个去找、去改。

- 触发范围：重命名/移动 `.md` 文件、图片等资源（`png`、`jpg`、`gif`、`svg`、`mp4` 等），以及移动文件夹。
- 默认会**先弹窗询问**再更新。弹窗里可勾选"总是这样做（不再询问）"：勾选后按"更新"表示以后自动更新、按"不更新"表示以后不再更新。
- 这个行为由设置项 `markdown.updateLinksOnFileMove.enabled` 控制，三个取值：
  - `prompt`（默认）：更新前弹窗询问。
  - `always`：静默自动更新。
  - `never`：关闭该功能。

> 锚点（`#小节`）、绝对路径、尖括号链接等细节都会被正确保留，行为与 VSCode 一致。

## 点击链接跳转文件和目录

文档里指向其它文件或目录的链接，可以直接点开——**源码模式**下 `Ctrl+点击`（macOS 用 `Cmd+点击`），**预览模式**下直接点击即可。路径可以写成：

- **相对路径**：`[配置](./config/settings.json)`，按"文档所在目录 → 工作区根目录"的顺序查找。
- **绝对路径**：`[vscode](D:/git_project/vscode)`，正斜杠 `/` 和反斜杠 `\` 都能识别，也可以指向工作区之外。

链接还能带上**行列位置**，打开后光标直接定位到那一行——在路径后加 `:行` 或 `:行:列`：

- `[跳到第 42 行](./src/main.ts:42)`
- `[跳到第 42 行第 8 列](./src/main.ts:42:8)`

点击后的行为按目标类型区分：

- **目标是文件**：在编辑器里打开。源码模式下 `Ctrl+点击` 在新标签打开；预览模式下 `Ctrl/Cmd+点击` 在侧边打开。
- **目标是目录**：用**新窗口**打开该文件夹。在**预览模式**下，`Ctrl+Alt+点击`（macOS `Cmd+Alt+点击`）改为在**当前窗口**打开该文件夹。

> 提示：源码模式受编辑器底层限制，无法区分是否按了 `Alt`，因此目录一律用新窗口打开；需要"当前窗口打开目录"时请在预览里用 `Ctrl+Alt+点击`。

## 关于回车和 Tab 键

在 Markdown 编辑模式下，`Enter`、`Tab`、`Shift+Tab` 有特殊行为：写列表时按 `Enter` 会自动续上下一个列表项，`Tab` / `Shift+Tab` 调整列表缩进层级。这些是编辑器自动帮你处理的，不需要你去命令面板找命令，正常敲键盘即可。

## 在预览中查找

在预览里找某段文字：让**预览获得焦点**（点一下预览区域），然后按 `Ctrl+F`，或运行"在预览中查找"。

- `F3` 跳到下一个匹配，`Shift+F3` 跳到上一个。
- `Escape` 关闭查找框。

## 用键盘跳转预览里的链接

预览里如果有很多链接，不想用鼠标一个个点，可以用键盘链接导航（类似 Vim 浏览器插件的用法）：

1. 让预览获得焦点。
2. 按 `F`，每个可见链接旁会浮出一个字母标签。
3. 敲出目标链接的标签字母，就会跟随那个链接。

相关按键：

- `Shift+F`：跟随链接时在侧边打开。
- `?`：显示这套键盘导航的快捷键说明。

除了 `F` 链接跳转，预览获得焦点时还能用 `j`/`k` 上下滚动、`d`/`u` 半屏翻页、`Space`/`Shift+Space` 整屏翻页、`gg`/`G` 跳到顶部/底部、`H`/`L` 后退/前进，数字前缀可重复（如 `3j` 向下滚三行）。完整列表按 `?` 查看。

> 提示：这套键盘导航同样适用于**文档中心**（帮助菜单 → "Documentation" 打开的内置文档）——它也是 Markdown 阅读界面，`F` 链接跳转、`j`/`k` 滚动、`Ctrl+F` 查找、`?` 帮助都一致可用。
