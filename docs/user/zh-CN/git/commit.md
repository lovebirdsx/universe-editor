# 提交你的改动

这是版本控制里最高频的一页。跟着走一遍"看改动 → 暂存 → 写提交信息 → 提交"，你就掌握了日常存档的全流程。本编辑器还有一个省心的特色：让 AI 帮你写[提交](../reference/glossary.md#提交--commit)信息。

## 看看你改了什么

提交之前，先看清楚自己动了什么。在[源代码管理](../reference/glossary.md#源代码管理scm)侧栏的"更改"分组里，点任意一个文件，就会打开它的[差异](../reference/glossary.md#差异--diff)（左边旧、右边新）。

对当前正在编辑的文件，也可以直接运行命令 **打开更改**（快捷键 `Shift+Alt+Y`）打开它的 diff。

## 暂存、取消暂存、放弃

看清改动后，挑出这次要存档的那些——这一步叫[暂存](../reference/glossary.md#暂存--stage)。把鼠标停在文件上，行内会出现按钮，也可以从命令面板运行对应命令：

| 你想做的 | 命令 | 说明 |
| --- | --- | --- |
| 把这个文件的改动加入下次提交 | **暂存更改** | 文件从"更改"移到"暂存的更改" |
| 把它撤出暂存区 | **取消暂存更改** | 反向操作，不删改动 |
| 丢掉这个文件的改动 | **放弃更改** | 把文件还原到上次提交的样子 |

批量操作对应 **暂存所有更改** / **取消暂存所有更改** / **放弃所有更改**。

> **务必小心："放弃更改"不可撤销。** 它会把文件还原到上一次提交的状态，你还没提交的改动会永久丢失。编辑器会先弹确认框（"要放弃'…'中的更改吗？此操作无法撤销。"），看清文件名再点确认。

## 写提交信息并提交

暂存好之后，在侧栏顶部的输入框写一句话，说明这次改了什么（输入框占位文字是 **"消息（Ctrl+Enter 提交）"**）。提交信息要让未来的你、或队友一眼看懂这次存档做了什么，比如"调低第 3 关 Boss 血量"。

写好后有两种方式提交：

- 直接在输入框按 `Ctrl+Enter`；
- 或点击 **提交** 按钮。

如果没写信息就提交，会提示"请先输入提交消息。"；没有任何改动可提交时，会提示"没有可提交的更改。"。

## 让 AI 生成提交信息

不知道提交信息怎么写？让 AI 替你写。输入框里有一个**闪光（sparkle）图标按钮**，对应命令 **Generate Commit Message**（生成提交信息）。点它，AI 会读取你当前的改动、自动写好一条提交信息填进输入框，你再按需修改即可。

> 命令面板里这条命令显示的是英文 **"Generate Commit Message"**，搜索时请用英文，不然找不到。

要用这个功能，需要先满足两个条件：

- **配好模型和供应商**：AI 得有可用的模型才能生成，配置方法见 [AI 供应商配置](../customization/ai-providers.md)。
- （可选）**指定专用模型**：从命令面板运行 **选择提交信息模型**，给"写提交信息"这件事单独挑一个模型。它和 [模型选择与花费](../ai-agent/models-and-cost.md) 里"按功能分配模型"中的**提交信息**是同一处配置，改哪边都一样。

如果想进一步定制，可在设置里配这两项（在设置里搜 `ai.commitMessage`）：

- `ai.commitMessage.instructions`：给 AI 的自定义提示，比如强制用中文、限定格式。
- `ai.commitMessage.maxDiffChars`：喂给 AI 的改动字数上限（默认 12000，超出会截断）。

## 提交的几种变体

除了普通的**提交**，还有几种常用变体，需要时再用：

- **提交（修正）**：把这次改动并进上一条提交、并改写它。适合"刚提交完发现漏了一点"。如果还没有暂存文件，它会先自动暂存当前更改；如果没有任何可修正的更改，会直接提示你。
- **提交并推送**：提交后顺手把它推送到[远程](../reference/glossary.md#源代码管理scm)。
- **提交并同步**：提交后先拉取远程更新、再推送（详见 [拉取、推送、储藏与冲突](./conflicts.md)）。
- **撤销上次提交**：把上一条提交退回成"未提交的改动"，改动还在，只是取消了那次存档。

提交按钮旁边有一个下拉入口 **提交操作...**，展开后可直接选普通提交、提交（修正）、提交并推送、提交并同步。

## 一个安全习惯

在让 AI [Agent](../reference/glossary.md#智能体--agent) 做大改之前，先提交一次干净的状态。这样无论 Agent 把项目改成什么样，你都能一键回退到这个干净起点。这一点在 [审阅并采纳 Agent 的改动](../ai-agent/reviewing-changes.md) 里也反复强调。
