# 拉取、推送、储藏与冲突

## 和远程同步

远程是仓库在云端 / 服务器上的副本。你的提交要"推"上去别人才看得到，别人的提交要"拉"下来你才拿得到。常用命令（在源代码管理侧栏 `…` 菜单，或命令面板）：

| 命令 | 作用 |
| --- | --- |
| **拉取** | 把远程的新[提交](../reference/glossary.md#提交--commit)拉下来并合并进本地 |
| **推送** | 把本地的提交推上远程 |
| **同步（拉取并推送）** | 一步到位：先拉取、再推送 |
| **获取** | 只查看远程有没有更新，**不**合并进来 |

还有几个变体，需要时再用：**拉取（变基）**、**拉取（自动储藏）**、**推送（强制）**、**推送到…**、**获取（清理）**。它们分别在 `…` 菜单的"拉取""推送""获取"子菜单里。

> **强制推送要格外当心。** **推送（强制）** 会用你本地的历史覆盖远程分支的历史，可能让别人已经推上去的提交丢失。编辑器会提醒"这将覆盖远程分支历史，可能导致他人的提交丢失"。除非你非常清楚后果，否则别用。

## 储藏：临时收起改动

改到一半，突然要去改别的分支、但又不想现在就提交这堆半成品？用**储藏**：它把当前改动临时收起来、让工作目录恢复干净，等回头再取回来。

| 命令 | 作用 |
| --- | --- |
| **储藏** | 把当前改动收进储藏，工作目录变干净 |
| **储藏（包含未跟踪文件）** | 连还没被 Git 跟踪的新文件一起收起 |
| **应用储藏…** | 取回某条储藏，但储藏仍保留 |
| **弹出储藏…** | 取回某条储藏，取回后删掉这条储藏 |
| **删除储藏…** | 直接丢弃某条储藏 |

## 解决合并冲突

当你拉取或[合并](./branches-and-merge.md#合并与变基)时，如果**两边都改了同一处内容**，Git 没法替你决定该听谁的，这处文件就进入**冲突**状态，需要你手动解决。有两条路径，取决于设置项 `git.mergeEditor`（默认开启三向合并编辑器）：

### 用合并编辑器（默认）

运行 **在合并编辑器中解决**，打开一个三向对比界面：一边是"当前"（你这边的版本）、一边是"传入"（对方的版本），中间是最终结果。逐处点选你要保留哪一边。

界面会提示还剩多少冲突（"{数量} 处冲突待解决"），全部处理完会提示"所有冲突已解决"。都解决后，运行 **完成合并** 收尾。

<!-- 截图：三向合并编辑器 -->

### 用内联标记

如果关掉了 `git.mergeEditor`，冲突会以 `<<<<<<<`、`=======`、`>>>>>>>` 这类标记**直接内联在文件里**。你在两段之间挑一段留下、删掉标记即可。在多处冲突之间跳转：

- **转到下一处合并冲突**：`Alt+F9`
- **转到上一处合并冲突**：`Shift+Alt+F9`

## 常见错误提示

同步时可能撞上这些提示，照着做即可：

- **"远程仓库有您本地没有的提交——请先拉取，或使用强制推送"**：远程比你新，先**拉取**再推。
- **"请先提交或储藏您的本地更改"**：手上有没处理的改动挡路，先**提交**或**储藏**它们再操作。

更多同步失败的排查，见 [常见问题排查](../reference/troubleshooting.md)。
