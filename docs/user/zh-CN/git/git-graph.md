# 提交历史图（Git 图谱）

Git 图谱是一个专门用来**可视化浏览提交历史和分支**的编辑器。想看清"项目是怎么一步步走到今天的"、或者直接在历史上对某个提交动手，这里是最佳入口。

## 打开 Git 图谱

两种方式：

- 命令面板运行 **View Git Graph**（查看 Git 图谱）；
- 或点源代码管理侧栏标题栏最左边的图谱图标。

> 命令面板里这条命令显示的是英文 **"View Git Graph"**（分类 "Git Graph"），搜索时请用英文。

打开后是一个编辑器标签页，标题为 **"Git 图谱"**。

## 图谱里能看到什么

- **每行一个[提交](../reference/glossary.md#提交--commit)**，左侧是连线图（画出分支怎么分叉、怎么合并），右侧有作者、提交、日期、描述等列。
- 最顶部有一行 **"未提交的更改"**，代表你当前还没存档的改动。
- **选中某个提交**，下方会列出它改了哪些文件，并按状态着色（已添加 / 已修改 / 已删除 / 已重命名等）。点击文件行会看这次改动的差异；行尾的 **打开文件** 会打开当前工作树里的源文件，如果文件已被删除或重命名，编辑器会提示原因。

顶部还有一些视图设置（进阶，扫一眼即可）：提交顺序（作者日期 / 日期 / 拓扑）、"仅跟随第一父提交"、以及"加载更多提交"。

## 搜索提交

图谱编辑器处于激活状态时，按 `Ctrl+F` 聚焦搜索框（占位文字"搜索提交…"），输入关键词快速定位某次提交。

> 命令面板里这条对应 **Focus Search**（聚焦搜索），且只在 Git 图谱编辑器激活时才生效。

## 显示或隐藏远程分支

想在图上看到[远程](../reference/glossary.md#源代码管理scm)分支，运行 **Toggle Remote Branches**（切换远程分支显示）来开关它们。图谱内也有对应的"显示远程分支""隐藏远程分支"入口。

> 命令面板里这条同样显示英文 **"Toggle Remote Branches"**，用英文搜索。

## 右键即操作

Git 图谱最强的地方，是可以直接在历史上对某个提交或分支**右键动手**。常用项包括：

- **检出此提交…**：把工作目录切到这个提交的状态。
- **在此处创建分支…** / **在此处创建标签…**：从这个提交开一条分支、或打一个标签。
- **合并到当前分支…**：把这个提交所在分支合进你当前的分支。
- **将当前分支变基到此提交…**、**将当前分支重置到此提交…**：进阶的历史整理操作。
- **拣选…**（cherry-pick）、**还原…**（revert）：单独把某个提交的改动挑过来、或撤销某次提交。
- **复制提交哈希** / **复制提交信息**：拷贝这次提交的编号或说明文字。
- **发送到 Agent Chat**（Send to Agent Chat）：把这次提交的哈希和提交信息发到 AI 会话的输入框，方便让 agent 结合这次提交回答问题。没有活动会话时会自动新建一个。

> 术语提示：这里的 **检出** 和源代码管理侧栏里的 **签出** 是同一个动作（切换到某个提交/分支），只是界面用词不同。

> **危险操作要看清确认框**：重置（reset --hard）、强制推送这类操作会改写历史、可能丢失内容，编辑器会弹确认框。看清楚提示再点确认，别手快。

## 图谱不可用时

如果当前项目文件夹不是 Git 仓库，图谱会提示 **"Git 图谱不可用 - 此文件夹是 git 仓库吗？"**。让项目先成为仓库的办法，见 [版本控制是什么，为什么需要它](./overview.md#让你的项目变成仓库)。
