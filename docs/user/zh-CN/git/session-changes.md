# 会话更改：AI 改了哪些文件

AI [Agent](../reference/glossary.md#智能体--agent) 会直接把改动写到你的磁盘上。**会话更改**视图，就是以 Git 的视角，把某一次 Agent 会话动过的所有文件聚成一处给你看。这一页讲怎么用它审阅，以及**真正动手（暂存 / 提交 / 回退）应该去哪做**。

## 会话更改和 Git 的改动有什么区别

[会话更改](../reference/glossary.md#会话更改)只盯着**这一次 Agent 会话**产出的文件改动，是"这次会话到底动了什么"的一本总账。而源代码管理侧栏的"更改"分组，盯的是**整个工作目录相对上次提交**的全部改动（不管改动是你手动改的还是 Agent 改的）。两者角度不同，会话更改帮你聚焦"AI 这一轮干了啥"。

## 打开会话更改视图

- 命令面板运行 **显示会话更改**（无默认快捷键）；
- 或打开会话编辑器标题栏的 `...` 菜单，选择 **显示会话更改**。

视图位于主侧边栏的"会话更改"容器里（图标是 diff / 一叠文件）。

关于空的状态：

- 当前没有活跃会话时，提示"没有活跃的 Agent 会话。"；
- 有会话但它还没改过文件时，提示"此会话尚未修改任何文件。"。

## 在视图里能做什么

- **单击**一个文件 = 预览它的[差异](../reference/glossary.md#差异--diff)（复用同一个预览标签，让你快速扫过多个文件而不堆一堆标签页）。
- **双击** = 把它钉成一个独立标签，长期打开。
- 每行还有 **打开文件** 入口；如果是 Markdown 文件，还多一个 **打开预览**。
- 右上角可以切换 **以列表查看** / **以树查看**：文件多时，树视图按目录层级组织更清楚。

## 重要：这个视图只用来看

会话更改视图**本身只用于查看**——它**没有**暂存、提交、放弃这些按钮。看完之后真正要动手时，请到别处：

- **要暂存 / 提交这些改动**：切到[源代码管理](../reference/glossary.md#源代码管理scm)侧栏，用 **暂存更改** + **提交**。手把手流程见 [提交你的改动](./commit.md)。
- **要回退这些改动**：用源代码管理侧栏的 **放弃更改**（注意不可撤销），或者从 [Git 图谱](./git-graph.md) 把项目重置到之前某个提交。

一句话记住：**会话更改视图看，源代码管理侧栏与 Git 图谱动手。**

## 一个安全习惯

Agent 是直接写盘的，会话更改就是"这一轮它动了什么"的总账。养成一个习惯：**让 Agent 大改之前，先干净地提交一次**。这样无论它改成什么样，你都能一键回到干净起点，审阅起来也更安心。
