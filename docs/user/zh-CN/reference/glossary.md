# 术语表

这里收录 Universe Editor 用户文档中反复出现的术语，给出统一译法和一句话解释。各册正文首次出现某个术语时，会链接到这里对应的词条。

> 说明：本表是术语基线版本，随参考册（07）定稿收口。译法一经确定，全站沿用；界面元素名称一律与你在软件里实际看到的中文一致。

## 工作区 / 项目

用"打开文件夹"载入编辑器的整个目录，就是你的工作区（也常说"项目"）。资源管理器里的文件树、全局搜索的范围、Git 集成，都以工作区为单位。

## 资源管理器

主侧边栏里的文件树视图，用来浏览、打开、管理工作区里的文件与文件夹。

## 命令面板

编辑器全部功能的统一入口。按 `Ctrl+Shift+P`（或 `F1`）打开，输入关键词就能找到并执行任意命令。记不住某个功能在哪，先打开命令面板搜一搜。详见 [命令面板](../getting-started/command-palette.md)。

## 辅助侧边栏

界面右侧的可选侧栏，用 `Ctrl+Alt+B` 显隐。放置 Agents、大纲、AI 调试等视图。（界面用词是"辅助侧边栏"，不是"次侧栏"。）

## 会话（Agent 会话）

你与 AI [智能体](#智能体--agent)的一轮完整对话上下文。每个会话有独立的历史、模型和改动记录，互不干扰。详见 [你的第一次 Agent 会话](../ai-agent/first-session.md)。

## 智能体 / Agent

能读写你项目里的文件、执行操作的 AI。区别于普通聊天 AI 只"告诉你怎么做"，智能体会"直接在你的项目里做完"，做完的改动你再逐条审阅。这是本编辑器的核心能力。界面上的面板名保留英文 "Agents"。

## 模型

[智能体](#智能体--agent)或编辑器 AI 功能背后用来思考的大语言模型。不同模型能力、速度、价格各异。注意区分 **Agent 模型**（作用于当前会话）与**平台 AI 模型**（作用于编辑器内置 AI 功能），详见 [模型选择与花费](../ai-agent/models-and-cost.md)。

## 模式（Agent 模式）

[智能体](#智能体--agent)的行为模式，可选项由当前 Agent 提供（例如偏只读咨询、或允许改文件等）。详见 [模式与思考等级](../ai-agent/modes-and-thinking.md)。

## 思考等级

[智能体](#智能体--agent)推理投入的深度。越高越细致，但也越慢、越贵。详见 [模式与思考等级](../ai-agent/modes-and-thinking.md)。

## 技能（Skill）

放在项目 `.claude/skills` 下的可复用能力包，让智能体按你的规范产出。在输入框敲 `/` 可看到可用技能。详见 [技能、记忆与 MCP](../ai-agent/skills-memory-mcp.md)。

## 记忆（Memory）

跨会话持久的项目上下文，记录在 `.claude/memory/MEMORY.md`（索引）及其分篇正文里。用来让智能体记住项目设定、术语译法、写作风格，省得每次重复解释。详见 [技能、记忆与 MCP](../ai-agent/skills-memory-mcp.md)。

## MCP

Model Context Protocol，一套让智能体接入外部工具与数据源的协议。详见 [技能、记忆与 MCP](../ai-agent/skills-memory-mcp.md)。

## 差异 / diff

改动前后的内容对比，左旧右新、按状态着色（新增 / 修改 / 删除）。审阅智能体或 Git 改动时都会看到 diff。

## 会话更改

某一次 [Agent 会话](#会话agent-会话)产生的文件改动集合，聚合在主侧边栏的"会话更改"视图里，逐文件可看 [diff](#差异--diff)。它是"这次会话动了什么"的总账。详见 [审阅并采纳 Agent 的改动](../ai-agent/reviewing-changes.md)。（界面用词是"会话更改"，不是"会话改动"。）

## 源代码管理（SCM）

编辑器的 Git 集成侧栏，管理暂存、提交、分支等版本控制操作。活动栏容器名显示英文缩写 "SCM"，视图标题为"源代码管理"。

## 暂存 / stage

把某处改动加入下一次提交的候选区。Git 的 stage 操作。

## 提交 / commit

把已暂存的改动记录成一次版本快照。Git 的 commit 操作。

## 工作树（worktree）

Git 的多工作目录特性，让同一个仓库同时在多个目录检出不同分支。注意不要和[工作区](#工作区--项目)（workspace）混淆。
