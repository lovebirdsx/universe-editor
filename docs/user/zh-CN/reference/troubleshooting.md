## 装不上 / 打不开

按顺序检查：

1. **系统要求**：确认是 Windows 10 或更高版本（64 位）。当前主要支持 Windows。
2. **被 SmartScreen 拦下**：首次运行弹出"Windows 已保护你的电脑"是正常的——当前产物未做代码签名。点**"更多信息"→"仍要运行"**放行。详见 [下载与安装 · 处理 Windows SmartScreen 警告](../getting-started/installation.md#处理-windows-smartscreen-警告)。
3. **杀毒软件拦截**：个别安全软件会对未签名程序更激进，必要时在其信任列表里放行。
4. **绿色版打不开**：如果用的是 `win-unpacked` 免安装目录，确认整个目录完整解压（不要只拷贝单个 exe），再运行里面的可执行文件。

装好后看到**欢迎页**就说明启动成功，详见 [下载与安装 · 首次启动](../getting-started/installation.md#首次启动)。

## Agent 连不上 / 无回复

Agent 发出去没反应、或报错时，按这个顺序查：

1. **是否配了供应商和模型**：这是最常见的原因。没有可用模型时会话发不出有效回复。去 [AI 供应商配置](../customization/ai-providers.md) 添加一个供应商分组、填好模型。
2. **密钥是否设置正确**：在 AI 设置里确认对应分组显示"已存储"密钥，可用"验证"按钮连一下、看能否列出模型。
3. **看具体报错**：内联补全等功能会给出明确的错误文案，对照下表判断根因——

   | 界面提示 | 含义 | 怎么办 |
   | --- | --- | --- |
   | API 密钥被拒绝（请检查密钥） | 密钥无效或填错 | 重新设置正确的密钥 |
   | 额度或余额已用尽 | 供应商账户额度 / 余额不足 | 去供应商侧充值或换分组 |
   | 请求频率受限，请稍后重试 | 触发了供应商限流 | 稍等片刻再试 |
   | 连接提供方时发生网络错误 | 网络不通 / 端点地址不对 | 检查网络与 Base URL |
   | 所选补全模型缺失 | 选中的模型已不存在 | 重新选择一个模型 |

4. **换个 Agent 试试**：内置有 Claude Code 和 Codex 两个 Agent，可用"选择 Agent 并新建会话…"切换验证。列表里标注"未安装（PATH 中找不到命令）"的 Agent 不可用。
5. **仍不行**：取一份日志便于反馈，见 [日志在哪、怎么取](#日志在哪怎么取)。

> 会话运行**中途**遇到限流、Agent 崩溃或长时间卡住时，编辑器会在输入框上方弹出状态条**自动重试 / 重连**并恢复被打断的那一轮，通常无需你手动干预；细节与手动重试见 [管理会话 · 中断自动恢复](../ai-agent/managing-sessions.md#中断自动恢复)。

相关：[你的第一次 Agent 会话 · 开始前先配好模型](../ai-agent/first-session.md#开始前先配好模型)。

## 内联补全不工作

内联补全（灰色幽灵文本）不出现时：

- **先配模型**：补全需要一个可用模型，没配时手动触发会弹提示引导你去选。见 [内联补全 · 开始前先配好模型](../editing/inline-completion.md#开始前先配好模型)。
- **是否被关掉**：运行"开关内联补全"确认它是开启状态。
- **手动触发**：把光标放进可编辑文本文件，按 `Alt+\` 主动求一条建议。
- **报错含义**：对照上一节的错误提示表。

详见 [内联补全（幽灵文本）](../editing/inline-completion.md)。

## 检查更新与升级

- **标题栏提示**：有更新可用或已下载待安装时，标题栏中间会出现一个带文字的醒目按钮（如"Update"、"Restart to update"），点击即可下载或重启安装；后台检查、下载中也会在此显示状态。
- **手动检查**：命令面板运行**检查更新**，编辑器会告诉你是"已是最新"、"发现新版本"还是"检查失败"。
- **发现新版本**：按提示下载；下载完成后会提示"版本 X 已准备好安装"，此时运行**重启以更新**（或点通知里的"立即重启"）完成安装。安装期间会短暂显示安装进度窗口，无需任何操作，装完自动重新打开编辑器。
- **检查失败**：多半是网络问题，稍后重试；持续失败可去 GitHub Releases 手动下载新版本覆盖安装。
- **自定义更新地址**：内网/自建更新源可运行命令面板的**打开更新配置 (JSON)**，在 `update-config.json` 里填 `updateUrl` 覆盖更新服务器（仅打包版生效）。

## 日志在哪、怎么取

反馈问题时附上日志能大大加快定位。取日志：

1. **看日志**：命令面板运行**开发人员：显示日志...**（`Ctrl+Shift+U`），挑一个日志文件，它会显示在输出底栏里。
2. **打开日志文件夹**：命令面板运行**开发人员：打开日志文件夹**，会在系统文件管理器里打开日志目录，方便你把日志文件打包发出去。

把相关日志连同"你做了什么、期望什么、实际发生了什么"一起反馈，最有帮助。

## Git 没反应

源代码管理侧栏空空的、Git 命令像没生效：

1. **当前目录不是 Git 仓库**：这是最常见的原因。版本控制只对仓库生效。本编辑器**没有内置"初始化仓库"命令**，需用系统 Git 执行 `git init`，或克隆一个已有远程仓库。详见 [版本控制 · 让你的项目变成仓库](../git/overview.md#让你的项目变成仓库)。
2. **[Git 图谱](../git/git-graph.md) 提示不可用**：出现"Git 图谱不可用 - 此文件夹是 git 仓库吗？"同样是因为当前目录不是仓库。
3. **没装系统 Git**：Git 集成依赖你系统里安装的 Git。若从未装过，先安装 Git 再重开项目。
4. **推送 / 拉取失败**：常见提示与处理见 [拉取、推送、储藏与冲突 · 常见错误提示](../git/conflicts.md#常见错误提示)，例如"请先拉取，或使用强制推送"、"请先提交或储藏您的本地更改"。
