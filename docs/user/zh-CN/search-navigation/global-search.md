## 先分清：搜内容 vs 找文件 vs 执行命令

开始前，先确认你要的是"全局搜索"。它和另外两个键盘入口很容易混：

| 快捷键 | 入口 | 用途 |
| --- | --- | --- |
| `Ctrl+Shift+F` | 搜索侧栏（全局搜索） | 在所有文件里搜文本内容，可替换 |
| `Ctrl+P` | 快速打开 | 按文件名跳转打开文件 |
| `Ctrl+Shift+P` | 命令面板 | 执行编辑器功能命令 |

> 记忆口诀：**搜内容用 `Ctrl+Shift+F`，找文件用 `Ctrl+P`，执行功能用 `Ctrl+Shift+P`。** 三者的完整对比见 [快速打开与命令面板](./quick-open.md)。

## 打开搜索侧栏

1. 按 `Ctrl+Shift+F`，或在[命令面板](../reference/glossary.md#命令面板)运行**在文件中查找**（`workbench.action.findInFiles`）。
2. 左侧打开"搜索"视图，输入关键词后立即在全项目里查找，结果按文件分组列出。
3. 点击任意一条匹配，跳转到对应文件的对应行。

省心的小细节：

- 如果打开搜索前在编辑器里选中了一段文字，它会被自动填进搜索框。
- 搜索侧栏已经聚焦时，再按一次 `Ctrl+Shift+F` 会把侧栏收起来。

## 快速搜索：临时找一下就跳走

只想临时找个关键词、看一眼就跳过去，不必展开整个侧栏时，用「快速搜索」：

- 按 `Ctrl+Q`，或在命令面板运行**快速搜索**（`workbench.action.quickTextSearch`）。
- 它以浮动选择器的形式出现，边输入边显示匹配，选中即跳转。

> 注意：快速搜索是 `Ctrl+Q`，不是 `Ctrl+Shift+Q`（后者是"聚焦大纲视图"，别按错）。

## 三个搜索选项

搜索框右侧有三个可切换的图标，控制怎么匹配：

| 选项 | 作用 |
| --- | --- |
| 区分大小写 | 打开后 `HP` 和 `hp` 视为不同 |
| 全字匹配 | 只匹配完整单词，`HP` 不会命中 `HPBar` |
| 使用正则表达式 | 用正则模式匹配，例如 `第\d+章` 匹配所有章节标题 |

> 正则表达式是一种描述文本模式的写法，适合"所有以某格式出现的内容"这类需求。不熟悉也没关系，日常用前两个选项就够了。

## 限定搜索范围

项目大的时候，把范围缩小能让结果更干净。点击搜索框旁的**切换搜索详情**（`search.toggleDetails`）展开范围设置：

- **要包含的文件**：只在这些文件里搜。例如填 `*.md` 只搜 Markdown，填 `docs/**` 只搜 `docs` 目录。
- **要排除的文件**：跳过这些文件。例如填 `*.json` 不搜 JSON。
- **使用排除设置和忽略文件**：打开后会尊重 `.gitignore` 等忽略规则，自动跳过不需要的文件。

## 跨文件替换

全局搜索最强的地方是能一次性改遍整个项目：

1. 点击搜索框左侧的**切换替换**（`search.toggleReplace`），展开替换输入框。
2. 在替换框里填入新内容。
3. 替换的方式：
   - **逐条替换**：把鼠标移到某条匹配行上，点击行旁出现的替换图标，只改这一处。
   - **按文件替换**：点击某个文件分组上的**替换文件中的所有匹配项**（`search.replaceAllInFile`），改掉这个文件里的全部匹配。

> 提示：批量替换是不可逆的大动作，建议改完先在编辑器里抽查几处，确认无误后再[提交改动](../git/commit.md)，这样万一改错还能回退。

## 管理搜索结果

结果列表顶部有几个按钮帮你整理视图：

- **以树查看** / **以列表查看**（`search.viewAsTree` / `search.viewAsList`）：在按文件夹分层和平铺列表之间切换。
- **清除搜索结果**（`search.clear`）：清空当前结果列表。
- **刷新**（`search.refresh`）：文件改动后重新执行一次搜索。
