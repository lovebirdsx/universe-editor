# 导航历史

跳到定义、点开搜索结果、来回翻文件之后，用两个快捷键沿着你走过的轨迹前后穿梭，一键回到出发点。就像浏览器的前进和后退。

## 像浏览器一样前进后退

用浏览器时，点开一个链接后想回到上一页，就按"后退"；后退多了想再往回走，就按"前进"。Universe Editor 的导航历史是一模一样的直觉——只不过这里"翻的页"是你跳转过的各个文件位置。

## 后退与前进

- **后退**：`Alt+Left`，或在[命令面板](../reference/glossary.md#命令面板)运行**后退**（`workbench.action.goBack`）。跳回上一个导航位置（可能在另一个文件里）。
- **前进**：`Alt+Right`，或在命令面板运行**前进**（`workbench.action.goForward`）。沿着刚才后退的路，再往回走一步。

> 提示：只有存在历史记录时这两个命令才可用。没有可后退/前进的位置时，命令面板里对应项显示为禁用，快捷键按下也没有反应——这是正常的。

## 清除导航历史

想清空已记录的跳转轨迹：

- 在命令面板运行**清除导航历史**（`workbench.action.clearHistory`）。
- **这个操作没有快捷键，只能从命令面板触发。**
- 清空后，`Alt+Left` / `Alt+Right` 在有新的跳转之前都不会再响应。

## 历史是怎么记录的

每次你打开文件、跳转到定义、点击搜索结果等操作，编辑器都会记下一条历史。`Alt+Left` 沿着这条链向回走，`Alt+Right` 向前走。

有一点值得知道：在同一个位置反复编辑不会新增历史记录——只有"跳到了别处"才算一次新的导航。

## 常见场景

导航历史最实用的时刻，就是你"深入去看了一眼、然后想回原地"：

- 按 `F12` 跳到某个 TypeScript 函数的[定义](./symbols-and-definitions.md)后，用 `Alt+Left` 回到刚才的调用处。
- 在[全局搜索](./global-search.md)结果里依次点开几个匹配文件查看后，用 `Alt+Left` 沿着打开的顺序一路退回来。
- 用 `Ctrl+T` 工作区符号跳转到另一个文件后，用 `Alt+Left` 回到出发的文件。

## 和标签页切换的区别

导航历史和「最近编辑器」（`Ctrl+Tab`）是两码事，别混用：

- **导航历史**（`Alt+Left` / `Alt+Right`）：沿你跳转过的**位置**轨迹前后走，能回到同一文件里的不同位置。
- **最近编辑器**（`Ctrl+Tab`）：在**已打开的标签页**之间按最近使用顺序切换。

`Ctrl+Tab` 的详细用法见 [快速打开与命令面板](./quick-open.md)。

## 本页涉及的命令

| 命令面板名称 | 快捷键 | 作用 |
| --- | --- | --- |
| 后退 | `Alt+Left` | 回到上一个导航位置 |
| 前进 | `Alt+Right` | 前往下一个导航位置 |
| 清除导航历史 | 无（仅命令面板） | 清空已记录的跳转轨迹 |
